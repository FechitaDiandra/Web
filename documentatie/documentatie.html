1. Introduction

Product Scope

The project aims to develop a web application that allows users to create custom forms for statistics and analysis. These forms can be tailored to specific entities or categories, enabling users to gather feedback and generate insights for their own purposes. The forms will capture feedback responses using the Plutchik emotion model.

In addition to creating forms, the web application will provide users with the ability to provide anonymous feedback for different entities (events, people, locations, products, services, etc.) using the Plutchik emotion model. Users will utilize intuitive forms to submit their emotional responses, which will be recorded and stored using a REST/GraphQL API.

2. Overall Description

Product Functions

  • Allow all users/page visitators to provide anonymous feedback for various entities.
  • Allow only logged users to create a form: the user can then publish it on the HomePage to get answers from other webpage users/visitators, or/and share it with the people they think would be the best fit to complete the form.
  • Generate statistics on evaluated entities based on recorded emotions after the collection period ends.
  • Users will be able to see their form history and access the statistics of any form anytime.
  • On the HomePage can be found the most recent forms published by users and it is actualized live.

User Classes and Characteristics

  • General users: These are individuals who provide anonymous feedback. They do not need to create an account or log in to the system. Their primary interaction with the system is through the feedback form.
  • Logged users: These are users who have registered and logged into the system. In addition to providing feedback, they can also create and manage their own feedback forms.
  • Administrators: They can manage all entities, users, forms and feedback in the system.

Operating Environment

  • Compatible with modern web browsers.
  • Supports various operating systems (Windows, macOS, Linux, etc.).

Design and Implementation Constraints

  • Utilizes REST/GraphQL API for backend.
  • Requires UTF-8 encoding for data transmission.

Assumptions and Dependencies

  • Users have access to a stable internet connection.
  • Users understand the Plutchik emotion model for providing feedback.

4. System Features

User Authentication and Authorization:

Implementing a secure login system that includes password encryption and session management. This could also include features like password recovery, email verification, and two-factor authentication.

Data Export

Users can export the data from their feedback forms in various formats (CSV, Excel, etc.) for further analysis.

Anonymous Feedback Submission

Users can provide feedback anonymously without requiring them to create an account or provide personal information. This can encourage more honest and unbiased feedback.

Emotion Selection

User-friendly interface for selecting emotions based on the Plutchik model.

Data Visualization

Implementing visually appealing data visualizations to represent feedback data, such as bar charts, pie charts, or line graphs. This can help users understand the data more intuitively and identify patterns or trends.

Social Sharing

Implementing social media sharing functionality to allow users to share feedback forms or specific responses on platforms like Facebook, Instagram, LinkedIn.

DataBases

The database will contain tables for forms, responses, and users. Each table will have a unique randomly generated ID for each record.

  • Forms Table: This will store the forms created by users. Each form will have its structure saved as JSON. The structure will include the form questions, response options, and any other relevant details.
  • Responses Table: This will store the responses received for each form. The responses will also be stored as JSON. Each response will be associated with the corresponding form through the forms ID.
  • Account Table: This will store user information. Each user will have a unique randomly generated ID when they create an account.

The database will allow for the creation, reading, updating, and deletion (CRUD) of records in these tables. It will also allow for complex queries to generate statistics based on the collected responses.
